# PSYCH QBANK — Инструкция для Claude Code

## Проект
UWorld-style банк вопросов по психиатрии. Целевая аудитория: практикующий врач-психиатр (не студент). Уровень: board-level (USMLE Step 2/3, Psychiatry Board).

## Архитектура
```
psych-qbank/
├── data/
│   ├── pretest/          ← спарсенные вопросы из PreTest (304 шт., исходный материал)
│   │   ├── evaluation.json
│   │   ├── mood.json
│   │   └── ...
│   ├── generated/        ← сгенерированные оригинальные UWorld-style вопросы
│   │   ├── psychopharm.json
│   │   ├── mood.json
│   │   └── ...
│   ├── manifest.json     ← реестр всех файлов и счётчиков
│   ├── domains.json      ← определения доменов
│   ├── topics.json       ← карта тем для генерации (seed data)
│   └── schema.json       ← JSON-схема для валидации
├── src/
│   └── app.jsx           ← UI (React, standalone)
├── scripts/
│   ├── validate.py       ← валидация JSON по схеме
│   └── stats.py          ← статистика по базе
├── CLAUDE.md             ← ЭТО ТЫ ЧИТАЕШЬ
└── README.md
```

## Формат вопросов (generated/*.json)

Каждый файл — массив объектов:

```json
{
  "id": "PH-042",
  "domain": "psychopharm",
  "tags": ["CYP-interactions", "SSRI", "warfarin"],
  "difficulty": 2,
  "vignette": "Клиническая виньетка на русском. 3-8 предложений. Конкретный пациент, возраст, пол, анамнез, текущая ситуация, лабораторные данные если релевантны.",
  "question": "Конкретный клинический вопрос.",
  "options": {
    "A": "Вариант ответа A",
    "B": "Вариант ответа B",
    "C": "Вариант ответа C",
    "D": "Вариант ответа D",
    "E": "Вариант ответа E"
  },
  "correct": "B",
  "explanations": {
    "A": "Почему A неправильно — механизм, в каком случае был бы правильным.",
    "B": "Почему B правильно — доказательная база, гайдлайн, механизм.",
    "C": "Почему C неправильно.",
    "D": "Почему D неправильно.",
    "E": "Почему E неправильно."
  },
  "pearl": "Одна ключевая мнемоническая фраза или правило.",
  "references": "Гайдлайны и источники (Maudsley, CANMAT, APA, NICE, DSM-5-TR)."
}
```

## Правила генерации

### Язык
- Виньетки, вопросы, объяснения — на РУССКОМ
- Терминология: DSM-5-TR / МКБ-10 двойная кодировка допустима
- References — на английском (названия гайдлайнов)

### Клинический уровень
- Целевая аудитория: практикующий врач-психиатр с NEI Master Psychopharmacology
- НЕ студенческий уровень ("что такое шизофрения?")
- Фокус на тактические решения: что назначить, когда менять, почему не другой вариант
- Дилеммы реальной практики: побочные vs эффективность, off-label, полипрагмазия

### Объяснения
- По КАЖДОМУ варианту (A–E), не только правильному
- Для неправильных: почему нет + в каком случае был бы правильным
- Механизм действия, не просто "не рекомендуется"
- NNT/NNH, effect sizes где возможно
- Конкретные исследования (STAR*D, CATIE, TADS, MTA, BALANCE)

### Источники (в порядке приоритета)
1. DSM-5-TR (APA, 2022)
2. Maudsley Prescribing Guidelines 15th Ed
3. CANMAT Guidelines (по расстройствам)
4. APA Practice Guidelines
5. NICE Guidelines (по номерам: CG178, NG87, etc.)
6. Stahl's Essential Psychopharmacology 5th Ed
7. Cochrane Reviews
8. Ключевые trials (STAR*D, CATIE, TADS, MTA, etc.)

### Юрисдикция
- Базовая: APA/международная
- Дополнительно для forensic: ЗУ "Про психіатричну допомогу" (Украина)
- Наказы МОЗ України где релевантно

### Difficulty
- 1: знание фактов (что такое NMS)
- 2: применение знаний (что назначить при X)
- 3: анализ + принятие решений при конкурирующих факторах (Y vs Z при коморбидности)

### ID-формат
- generated: `{DOMAIN_PREFIX}-{NNN}` (PH-001, MD-023, PS-005)
- pretest: `PT-{DOMAIN}-{NNN}` (уже присвоены, не менять)

## Рабочий процесс генерации

1. Прочитай `data/topics.json` — там карта тем по доменам
2. Прочитай пример из `data/generated/psychopharm.json` — это твой template
3. Генерируй батчами по 10-20 вопросов в один файл
4. После генерации запусти `python scripts/validate.py` для проверки
5. Commit с описательным сообщением: `feat(mood): add 15 questions on bipolar depression`

## Запрещено

- Копировать вопросы из PreTest дословно (они в data/pretest/ как reference)
- Использовать DSM-IV терминологию (только DSM-5-TR)
- Генерировать вопросы уровня "что из перечисленного — симптом депрессии"
- Объяснения типа "не рекомендуется" без механизма почему
- Оставлять поля пустыми (все поля обязательны)
